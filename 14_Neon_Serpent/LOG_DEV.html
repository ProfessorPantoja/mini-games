visão · multiplayer arquitetura

Como eu faria o online (recomendação)

  • Ordem de implementação: (1) single polido → (2) duo local same-screen → (3) online. Duo local prova regras de 2 jogadores sem rede.
  • Modelo de netcode: autoritativo no servidor (servidor simula o step da snake; clientes só enviam input). Snake é discreto em grade — perfeito para lockstep por tick.
  • Stack leve: Node + WebSocket (ou Cloudflare Durable Objects / PartyKit se quiser hospedar barato). Sala de 2–4, matchmaking simples por código de sala.
  • Tick: ex. 8–12 Hz de estado; inputs com sequência; reconciliação leve no cliente (predição só da própria cobra).
  • Não faça primeiro: rollback complexo tipo GGPO — overkill para snake em grid. Comece com “servidor manda estado a cada step”.
  • Escopo online MVP: 1 arena, 2 jogadores, sem campanha, score + “último vivo” ou “primeiro a N orbes”. Campanha online fica para depois.
  • Anti-cheat básico: servidor valida movimento; cliente nunca decide morte do oponente.
